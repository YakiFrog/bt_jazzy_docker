# トラブルシューティング

## よくあるエラーと対処法

### 1. `LogicError: The port ... can not be converted to float`
*   **症状**: `run_bt` を実行した直後にクラッシュし、このメッセージが表示される。
*   **原因**: XML ファイル内で、そのノードの引数（ポート）の値が空っぽ（空文字列）になっています。
*   **解決策**: Groot2 で該当する XML を開き、ノードのパラメータ欄に数値を入力して保存してください。

### 2. `Action server ... is not reachable`
*   **症状**: 実行してもノードが動かず、この警告が出る。
*   **原因**: 技能サーバー（Python ノード）が起動していないか、通信が届いていません。
*   **解決策**:
    1.  `run_logic` が別のターミナルで実行されているか確認。
    2.  `node_manager` でノードを新規作成した場合は、`build` を行ってから `run_logic` を再起動したか確認。

### 3. `node_manager` でファイルを書き換えられない
*   **症状**: ノードの削除や生成時に Permission エラーや File Not Found が出る。
*   **原因**: 指定している `Trees Directory` が間違っているか、Docker 外でファイルを編集してパーミッションが変わっている可能性があります。
*   **解決策**: `node_manager` 上部のディレクトリ設定を確認してください。

---

## 開発のヒント
*   **困ったらログを見る**: 技能ノードの詳しい動きは `run_logic` を実行しているターミナルの出力に表示されます。
*   **変更が反映されない**: Python のロジックを変更した後は、必ず `run_logic` を一度止めてから再起動してください（`Ctrl+C` で止まります）。
