# 新しい振る舞い（Action）の開発フロー

このドキュメントでは、新しいロボットの動作を追加する際の手順と、設計思想について解説します。

---

## 1. 設計思想：なぜ Action なのか？

Behavior Tree (BT) において、各ノードはロボットの **「状態 (State)」** を表します。ROS 2 の Action 通信を使う理由は、BT の要求する以下の特性を満たすためです。

- **非同期性**: 動作中に BT をブロックせず、「実行中 (RUNNING)」という状態を維持できる。
- **中断可能性 (Preemption)**: より優先度の高いイベントが発生した際、実行中の動作を安全にキャンセルできる。
- **フィードバック**: 動作の進捗を確認し、それに応じた判断ができる。

---

## 2. 開発ステップ（5 つのステップ）

新しい動作を追加する際は、以下のフローで進めます。

### ステップ 1：通信（Action）の定義
`src/bt_msgs/action/` に新しい `.action` ファイルを作成します。

```text
# 例: MoveToTarget.action
float32 x
float32 y
---
bool success
---
float32 distance_to_target
```

### ステップ 2：ロジック（Python）の実装
`src/bt_python_logic` 内で、実際の動きを担当する Action Server を作成します。
- センサー値の読み取りや、モーター制御などの「具体的な仕事」はすべてここで行います。
- `goal_handle.publish_feedback()` を使って進捗を報告するようにします。

### ステップ 3：BT ノード（C++）の実装
`src/bt_example/src/main.cpp` に、Python 側を呼び出すためのノードを作成します。
- `RosActionNode` を継承して作成します。
- XML からパラメータ（目標値など）を受け取るためのポートを定義します。

### ステップ 4：工場の登録 (C++)
`main.cpp` の `main` 関数内で、新しいノードを `BehaviorTreeFactory` に登録します。

```cpp
factory.registerNodeType<MoveToTargetAction>("MoveToTarget", params);
```

### ステップ 5：ツリーのデザイン (XML / Groot2)
XML ファイルを編集、または Groot2 を使って新しいノードを配置します。
- 順次実行なら `Sequence`、条件分岐なら `Fallback` や `IfThenElse` を組み合わせて「知能」を組み立てます。

---

## 3. 開発のコツ

- **ノードはシンプルに**: 一つのノードに複雑な条件分岐を詰め込まず、「前進する」「挨拶する」といった単純なアクションに分割してください。
- **再利用性を考える**: 汎用的なアクション（「特定の場所へ行く」など）を作っておけば、XML を書き換えるだけで様々なミッションに対応できるようになります。
- **Groot2 で監視する**: 実行中は常に Groot2 で接続し、どのノードで止まっているか、どこで失敗しているかを視覚的に確認しながら開発してください。
